# MAWA Anti-Patterns

Avoid these patterns:

- Treating MAWA as a generic framework generator.
- Asking the human to copy/paste the next prompt during a continuous run.
- Creating competing artifact paths such as both `docs/specs` and `specs`.
- Keeping a stale local copy of adapter docs instead of reading the adapter's canonical `source_url`.
- Installing dependencies silently during setup.
- Mixing UI adapter rules into core governance.
- Implementing from a design handoff without validating the approved layout.
- Writing code for a module before its module spec is approved.
